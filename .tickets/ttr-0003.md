# Configuration Parsing

Implement parsing of `.tickets/sync.toml` configuration file.

## Description

Parse the sync configuration file with the following schema:

```toml
[github]
repo = "owner/repo"              # Required
project = "Project Name"         # Optional
assignee = "username"            # Optional

[mapping]
type_field = "Type"              # Optional, default "Type"

[mapping.type]
bug = "Bug"
feature = "Feature"
task = "Task"
epic = "Epic"
chore = "Chore"

[labels]
sync_tags = true                 # Optional, default true
create_missing = true            # Optional, default true
```

Implement in `src/config.rs`:
- `Config` struct with serde derive
- `Config::load()` to find and parse config file
- Walk parent directories to find `.tickets/sync.toml` (like the ticket tool does)
- Validation of required fields
- Sensible defaults for optional fields

## Acceptance Criteria

- [ ] Config struct matches schema
- [ ] Parses valid TOML files correctly
- [ ] Returns clear error for missing required fields
- [ ] Applies defaults for optional fields
- [ ] Searches parent directories for .tickets/sync.toml
