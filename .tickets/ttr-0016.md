# Init Command

Implement `ttr init` for interactive configuration setup.

## Description

Add an init command that creates `.tickets/sync.toml` with sensible defaults and optional interactivity.

### Behavior

```
$ ttr init

Creating .tickets/sync.toml...

GitHub repository (owner/repo): myorg/myrepo
GitHub Project name (optional): Q1 Sprint
Default assignee (optional): acmyers

Created .tickets/sync.toml

Next steps:
  1. Review and customize the config
  2. Run 'ttr push' to sync tickets
```

### Non-Interactive Mode

```
$ ttr init --repo myorg/myrepo --project "Q1 Sprint" --assignee acmyers
```

### Default Config Template

```toml
[github]
repo = "owner/repo"
# project = "Project Name"
# assignee = "username"

[mapping]
type_field = "Type"

[mapping.type]
bug = "Bug"
feature = "Feature"
task = "Task"
epic = "Epic"
chore = "Chore"

[labels]
sync_tags = true
create_missing = true
```

## Acceptance Criteria

- [ ] Creates .tickets directory if needed
- [ ] Creates sync.toml with defaults
- [ ] Interactive prompts for required values
- [ ] Non-interactive mode with flags
- [ ] Doesn't overwrite existing config (error with --force to override)
